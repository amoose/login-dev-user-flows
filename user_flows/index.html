/ user flows

       SP-initiated authentication with login.gov
              with a valid SP
                     prompts the user to create an account or sign in »
                     when choosing Create Account
                            displays an interstitial page with information »
                            prompts for email address »
                            with a valid email address submitted
                                   informs the user to check email »
                                   with a confirmed email address
                                          prompts the user for a password »
                                          with a valid password
                                                 prompts the user to configure 2FA »
                                                 with a valid phone number
                                                        with SMS delivery
                                                               prompts for OTP »
                                                        with Voice delivery
                                                               prompts for OTP »
                     when choosing to sign in
                            with valid credentials entered
                                   prompts for 2FA delivery method »
                                   with SMS OTP selected (default)
                                          prompts for OTP verification »
                                          with valid OTP confirmation
                            without a valid username and password
                                   when choosing "Forgot your password?"
                                          prompts for my email address »
                                          with not_a_real_email_dot.com submitted
                                                 displays a useful error »
       Visitors requesting login.gov directly
              when visiting the homepage
                     loads the home page »
                     showing the password
                            allows me to see my password »
                     when attempting to sign in
                            with a valid account
                                   sends OTP via previously chosen method »
                                   with a valid OTP
                                          redirects to profile »
                                   with an invalid OTP submitted
                                          displays a useful error »
                                          with a second invalid OTP submitted
                                                 displays a useful error »
                                                 with a third invalid OTP submitted
                                                        displays a useful error and locks the user account »
                            without valid credentials
                                   displays a useful error message »
                     when choosing create account
                            informs the user about login.gov »
                            when creating account with valid email
                                   notifies user to check email »
                                   when confirming email
                                          prompts user to set password »
                            when attempting with an invalid email
                                   informs the user to try again »
